基本資料路徑：

```text
PC
→ Instruction Memory
→ Instruction Register
→ Decode
→ Register File
→ ALU / Writeback
→ Register File
```

Branch path：

```text
Register data == 0
→ BEQZ condition
→ signed immediate / offset
→ PC branch selection
```

## CPU v2 指令

目前 8-bit instruction 使用 2-bit opcode，支援：

| 指令 | 功能 | 目前狀態 |
|---|---|---|
| `ADD` | 兩暫存器相加並寫回 | RTL 已整合；Phase 0-1 手推驗收通過 |
| `ADDI` | 暫存器加 signed immediate | RTL 已整合 |
| `BEQZ` | 暫存器為 0 時相對分支 | RTL 已整合 |
| `CP` | 暫存器資料複製 | RTL 已整合 |

其他特性：

- [x] 8-bit Program Counter
- [x] 8 個 8-bit registers
- [x] R0 固定讀出為 0，禁止寫入
- [x] 3-bit immediate sign extension
- [x] 兩狀態 `FETCH`／`EXECUTE`

因此目前 testbench 屬於 **waveform-observation testbench**，尚不是完整 self-checking testbench。

## Instruction Memory

FPGA 版本：

```text
_8bitscpu/v2/fpga/cpu/instruction_memory.v
```

已使用：

```verilog
$readmemb("program.mem", mem);
```

並已有一份測試程式：

```text
_8bitscpu/v2/fpga/cpu/program.mem
```

但核心模擬版本：

```text
_8bitscpu/v2/cpu/instruction_memory.v
```

仍採用不同的初始化方式，與 FPGA 版本不一致。後續應統一，避免模擬與上板使用不同程式或不同 RTL。

## 已知版本分歧

目前 `_8bitscpu/v2/cpu/` 與 `_8bitscpu/v2/fpga/cpu/` 存在部分重複模組。

可能造成：

- 修改其中一份，另一份未同步
- testbench 測到的不是上板版本
- Instruction Memory 初始化方式不同
- 修正 bug 後只在其中一份生效

這是目前復健階段需要優先整理的工程問題。

# 六、長期系統目標

最終希望完成：

```text
CPU
├─ Instruction Memory
├─ Data RAM
├─ System Bus
├─ Memory-mapped UART
├─ Timer / GPIO（可選）
└─ GPU command interface

GPU / Display Subsystem
├─ Command FIFO
├─ 2D drawing engine
├─ VRAM / Framebuffer
└─ VGA Timing Controller

Input
├─ UART RX
└─ Keyboard Controller（進階）
```

設計原則：

- CPU 不直接產生 VGA pixel timing。
- UART、GPU、VGA 與鍵盤皆為獨立模組／周邊。
- CPU 透過 System Bus 與 memory-mapped registers 控制周邊。
- GPU 負責修改 VRAM；VGA Controller 持續讀取 VRAM 輸出畫面。
- 第一版優先使用低解析度 BRAM framebuffer，再考慮外部 SDRAM。
- 每一層先獨立 testbench，再做系統整合。
