### Phase 0-3：`BEQZ` 指令格式、分支條件與完整手推

唯一目標：能獨立完成一條 `BEQZ` 的編碼、分支條件判斷，以及 branch taken／not taken 的 FETCH／EXECUTE 時序手推。

預計驗收重點：

- 第三欄位在 `BEQZ` 中作為 signed branch offset。
- `branch_en` 啟用零值比較路徑。
- `reg_write = 0`，BEQZ 不寫回 Register File。
- 能分辨 branch taken 與 not taken 時的 PC 更新結果。
- 能依目前 RTL 說明 branch target 使用的 PC 基準。
- 能正確說明 IR、PC 與 Register File 的更新 edge。
